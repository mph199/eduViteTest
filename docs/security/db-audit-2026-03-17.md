## 2. Kritische Befunde (KRITISCH)

### K1: Kein Loeschkonzept fuer booking_requests
- **Quelle:** DB-Analyst
- **Tabelle:** `booking_requests`
- **Problem:** Kein DELETE-Endpunkt, keine Anonymisierung. parent_name, student_name, email, message bleiben unbegrenzt.
- **DSGVO:** Art. 5 Abs. 1 lit. e (Speicherbegrenzung), Art. 17 (Recht auf Loeschung)
- **Empfehlung:** Anonymisierungs-Migration + automatisches Cleanup nach Event-Abschluss

### K2: Art.-9-Daten ohne Schutz in SSW/BL
- **Quelle:** DB-Analyst
- **Tabellen:** `ssw_appointments.concern/notes`, `bl_appointments.concern/notes`
- **Problem:** Psychosoziale Beratungsanliegen (Freitext) ohne Verschluesselung, Zugriffsprotokollierung oder Aufbewahrungsfristen
- **DSGVO:** Art. 9 (besondere Kategorien), Art. 35 (Folgenabschaetzung erforderlich)
- **Empfehlung:** Aufbewahrungsfrist definieren, Zugriff auf zugewiesenen Berater beschraenken, Verschluesselung erwaegen

### K3: Cancel in SSW/BL loescht PII nicht
- **Quelle:** DB-Analyst
- **Dateien:** `backend/modules/schulsozialarbeit/routes/counselor.js`, `backend/modules/beratungslehrer/routes/counselor.js`
- **Problem:** Cancel setzt nur `status='cancelled'` – student_name, email, phone, concern, notes bleiben erhalten
- **Empfehlung:** Bei Cancel: PII-Felder auf NULL setzen (analog `cancelBookingAdmin()` bei slots)

## 3. Hohe Befunde (HOCH)

### H1: Transaktion ueber Pool-Wrapper (Connection-Isolation fehlt)
- **Quelle:** Konsistenzpruefer, Hygieniker
- **Datei:** `backend/routes/admin/userRoutes.js:107`
- **Problem:** `query('BEGIN')` / `query('COMMIT')` ueber Pool-Wrapper – BEGIN und folgende Statements koennen auf verschiedenen Connections landen
- **Empfehlung:** Auf `getClient()` + `client.query()` umstellen (wie `teacherRoutes.js:566`)

### H2: Connection-Leak in eventsRoutes
- **Quelle:** Konsistenzpruefer
- **Datei:** `backend/routes/admin/eventsRoutes.js:199-201`
- **Problem:** `client.query('ROLLBACK')` im catch, aber kein `client.release()` im finally-Block
- **Empfehlung:** `finally { client.release() }` ergaenzen

### H3: rejectUnauthorized: false bei DB-SSL
- **Quelle:** Waechter
- **Datei:** `backend/config/db.js:42`
- **Problem:** Deaktiviert Zertifikatsvalidierung, ermoeglicht MITM-Angriffe
- **Empfehlung:** CA-Zertifikat explizit angeben oder nur fuer bekannte Self-Signed-Setups zulassen

### H4: Klartext verification_token in slots
- **Quelle:** DB-Analyst
- **Tabelle:** `slots.verification_token`
- **Problem:** Migration 011 fuehrte Hash ein, aber Original-Klartext-Spalte besteht weiter
- **Empfehlung:** Migration 034: `DROP COLUMN verification_token`, Dual-Lookup im Code entfernen

### H5: Kein RLS auf users, ssw/bl-Tabellen
- **Quelle:** DB-Analyst
- **Problem:** Nur `teachers`, `slots`, `feedback`, `events` haben RLS. `users`, `ssw_*`, `bl_*` nicht.
- **Empfehlung:** RLS aktivieren + restriktive Policies

### H6: Fehlende Indizes auf booking_requests
- **Quelle:** DB-Analyst
- **Problem:** Kein Index auf `email` und `verification_token_hash`
- **Empfehlung:** `CREATE INDEX IF NOT EXISTS idx_booking_requests_email/token_hash`

### H7: generate-slots dreifach dupliziert
- **Quelle:** Hygieniker
- **Dateien:** `bl/counselor.js:169-259`, `ssw/counselor.js:69-162`, `counselorAdminRoutes.js:374-473`
- **Problem:** ~180 Zeilen nahezu identisch, nur Tabellenname unterschiedlich
- **Empfehlung:** In `shared/generateCounselorSlots.js` extrahieren
