```
SSW/BL:           Formular → ssw/bl_appointments (PII, KEINE Art.9-Daten mehr)
                  Cancel: status='cancelled'

                  Admin-DELETE: physische Loeschung moeglich

Consent-Audit:    consent_receipts (append-only, ip_address, user_agent)
                  NICHT loeschbar (Nachweispflicht Art. 7 Abs. 1 DSGVO)

Security-Audit:   audit_log (user_id, ip_address)
                  Automatische Rotation nach 730 Tagen (retention-cleanup.js)
```

## 7. Technische Details

### Extensions
- `pgcrypto` 1.3 (Passwort-Hashing, Token-Generierung)
- `plpgsql` 1.0

### Funktionen (App-spezifisch)
- `anonymize_booking_request(request_id INT)` – Setzt PII-Felder auf NULL, restricted=TRUE
- `anonymize_booking_requests(event_id INT)` – Anonymisiert alle Requests eines Events
- `anonymize_booking_requests_by_email(email TEXT)` – Anonymisiert alle Requests fuer eine E-Mail (NEU Mig 047)
- `reset_slot_sequence()` – Setzt Slot-ID-Sequenz zurueck
- `reset_teacher_sequence()` – Setzt Teacher-ID-Sequenz zurueck
- `set_updated_at()` – Trigger-Funktion fuer `users.updated_at`

### Trigger
- `trg_users_set_updated_at` – BEFORE UPDATE auf `users` → `set_updated_at()`

### Naechste freie Migration: 049

> Migration 048: `048_fix_default_admin_force_password.sql` – Setzt `force_password_change = TRUE` fuer Default-Admin `Start` (2026-03-19)
